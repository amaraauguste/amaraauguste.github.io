  • June: 30 days
  • July: 31 days
  • August: 31 days
  • September: 30 days

Every year that is exactly divisible by four is a leap year, except for years that are exactly divisible by 100, but these centurial years are leap years if they are exactly divisible by 400

So to evaluate if whether or not it is a leap year, we need to check whether:

  1. It is evenly divisible by 100
  2. If it is divisible by 100, then it should also be divisible by 400

b) Write a Java program that prompts the user to input an integer and then outputs the number with the digits reversed. For example, if the input is 1234, the output should be 4321.

Here is a step-by-step walkthrough to help you outline a solution:

  • First, prompt the user for the number that they want to reverse

  • Create any necessary variables, we will need two:
    1. a variable initialized to 0 to hold the reversed number (let's call it reverseNum)
    2. a variable to hold the temporary values needed during calculation also initialized to 0 (let's call it rem)

  • At the beginning of each loop, we'll multiply reverseNum by 10.

  • Now, let's take our original number and divide it by 10 using the modulus operator. This is to find a single-digit remainder which equals the current last digit of nums, we can hold that value in our temporary variable rem.

  • Then subtract rem from num and divide the result by 10. And now we're left with the same integer, but one digit less.

    (Why are we dividing and multiplying numbers by 10?

    It's because we're replicating decimal place values where each place has a value of times ten from right to left.

    Dividing by 10 eliminates the last zero in num, which then gives us access to the next digit that ends up in the ones place.

    Multiplying reverseNum by 10 makes room in the ones place where we can place the digit we popped off from num.)

  • Next, it's time to "push" the "popped" digit from num by taking the remainder (that we saved in the variable rem) and adding it to reverseNum.

  • Now, we've completed one cycle of this process.

  • Repeat until num's value dwindles to 0 and there are no more digits to 'pop' or 'push'.

    After the reversal of digits is complete, we can finally print out reverseNum.

a) Write a Java program that prompts the user to enter the number whose factorial they wish to compute and then calculates the factorial of that number.

The factorial of any number n:

  • is represented by n!
  • and is equal to 1*2*3*....*(n-1)*n
Example: the factorial of 5 is represented as 5!

And, 5! = 5 * 4 * 3 * 2 * 1 = 120

Note: The value of 0! is 1, according to the convention for an empty product.
